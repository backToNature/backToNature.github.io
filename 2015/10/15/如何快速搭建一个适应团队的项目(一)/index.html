如何快速搭建一个适应团队的项目(一)

发表于 2015-10-15   |   分类于 web前端

本文论述如何搭建一个前端项目，当中会涉及设计模式与一些前言框架的应用，当然最重要的还是讨论如何快速搭建一个项目。我们从一个最简单的例子出发:

表格问题

如何搭建一个简易的架子来开发这个页面呢？

如何让这个架子更容易维护与扩展呢？

下面我们便围绕这两个问题来解决这个前端程序员几乎写到吐的场景。

MV*

这里用MV*(MVC、MVP、MVVM)的模式来做这件事情,关于这些模式这里就不多做讨论了(不了解这些模式的可以仔细阅读一下这个话题——你对MVC、MVP、MVVM 三种组合模式分别有什么样的理解？)。这种模式目前流行的前端框架有很多，例如:backbone.js,riot.js,angularjs，这里用backbone来举例。

方案一

建立一个Model,model.js代码如下:

var Model = Backbone.Model.extend({
    defaults: {
        // Model中的初始数据
        list: [
            /*
            {
                id: 502,
                time: '1992-12-24',
                name: '小豪豪'
            }...
            */
        ]
    },
    // Model初始化会执行initialize函数
    initialize: function () {
        var _this = this;
        // code here
    },
    search: function (id, time) {
        var _this = this;
        $.ajax({
            url: 'searchUrl',
            data: {
                time: time,
                id: id
            },
            type: 'post',
            dataType:'json',
            success: function (data) {
                if (data.status === 'success') {
                    // 这里的set操作是重点
                    _this.set('list', data);
                }
            }
        });
    },
    add: function (time, name) {...},
    del: function (id) {...}
});

Model中的list存储了业务数据,而页面上的所有交互都是围绕业务数据进行的。这里我们可以使用observer模式，对Model中的数据进行观察，如果Model数据发生了变化，就重新渲染表格。presenter.js代码如下:

// 如果Model数据发生改变，就重新渲染表格
Model.on('change', function () {

    var _this = this,
        data = this.get('list');

    // 根据模板tpl渲染数据
    $table.render(tpl, data);
});

// 交互逻辑

$searchButton.on('click', function (data) {

    var id = $idInput.val(),
        time = $timeInput.val();
    Model.search(id, time);
});

$addButton.on('click', function (data) {
    ...
    Model.add(name, time);
});

$delButton.on('click', function (data) {...});

到这里，其实主要的逻辑已经完成了。这样，我们就不用关心在这个页面中有多少和表格相关的交互操作了，只需要操作Model的数据就行了。

5分钟思考一下: 如果现在要求做出查询数据操作后给出操作状态提示，怎么来更新代码?如果又要求添加数据操作后给出操作状态提示，怎么来更新代码？

方案二

我们要想给出操作状态提示就得在ajax回调里拿到后端返回的操作状态，但ajax写在Model里，严格来说的话，一般是不允许在Model里直接操作view的。所以如果遇到这种需求，我们可以采用observer模式的变种Pub/Sub(发布者/订阅者)模式，我们修改Model中的search方法，代码如下:

search: function (id, time) {
    var _this = this;
    $.ajax({
        url: 'searchUrl',
        data: {
            time: time,
            id: id
        },
        type: 'post',
        dataType:'json',
        success: function (data) {
            Backbone.Events.trigger('SEARCH_CALLBACK', data);
            if (data.status === 'success') {
                // 添加一个事件发布者
                _this.set('list', data);

            }
        }
    });
},

然后我们就可以在presenter.js加上如下代码:

// 监听SEARCH_CALLBACK事件
Backbone.Events.on('SEARCH_CALLBACK', function (data) {
    if (data.status === 'success') {
        // 给出操作成功提示
    } else {
        // 给出操作失败提示
    }
});

这样的话，既避免了view侵入model，又可以更方便地做更多的交互。
